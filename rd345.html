Rubber Ducky 345

This is a 24-hour hackathon project coded using Java and Processing . The game makes use of Google's QuickDraw public dataset of hundreds of thousands of doodles of objects (345 in total!) from the hands of countless users throughout the world.

The idea is that the protagonist of the game, rubber ducky, travels randomly to different worlds, where each world has a different theme indicated by a noun (e.g. "violin", "horse", etc.)

The game searches up the keyword on Google with web requests and gets a list of images from Google' search result. One of those images is displayed as the background of the current world, while the other images are used for calculating a theme color for the noun. This is achieved by looping through all the pixels of the images and finding out the average color of the images.
